# LeetCode 笔记

## 项目结构

笔记统一存放在 `leetcode/problems/oj/leetcode/<题号>-<英文短横标题>.md`。

## 笔记格式规范

每篇笔记按以下顺序组织：

### 1. 标签行

单独一行：`#标签1 #标签2`，无需前导空格。常用标签：
`#哈希表` `#字符串` `#滑动窗口` `#栈` `#链表` `#二分` `#前缀和` `#差分数组` `#优先队列` `#双指针` `#动态规划` `#背包问题` `#数论` `#区间调度` `#计数` `#最优化` `#存在性` `#二维` `#容斥` `#组合`

### 2. 题目链接

Markdown 链接格式。cn 域名用中文标题 + `力扣（LeetCode）`，com 域名用英文标题 + `LeetCode`。

### 3. 代码版本

每个版本用 `>标签` 作为标题，紧跟代码块（带语言标识 `python` 或 `c++`）。

**常见标签对照：**

| 你说的情况 | 标签写法 |
|---|---|
| 第一版写错了 / 边界不对 | `>第一版wrong` 或 `>错误版` |
| 超时 | `>超时` |
| 有 bug / 报错 | `>报错版` |
| 通过了 | `>第一版通过` 或 `>通过` |
| 手写 / 自己写的 | `>手写版` 或 `>我的题解` |
| GPT/AI 帮改 | `>GPT改进` 或 `>GPT版` |
| Gemini 帮改 | `>Gemini版` 或 `>Gemini标准版` |
| 提示后通过 | `>提示通过版` |
| 修正 / 改正后 | `>改正版` 或 `>修正版` |
| 某日期写的 | `>2026年3月8日` |
| 最终版 | `>最终通过` 或 `>通过` |

### 4. 方法论部分（可选）

代码版本之后可添加：错误分析、口诀记忆、对比表格、学习总结等。可用 `###` 标题、表格、列表、引用块等格式。

### 5. 下一版占位

始终以空代码块结尾：
```
>下一版
```python

```
```

## 文件命名

`<题号>-<英文短横标题>.md`，如 `1-two-sum.md`、`209-minimum-size-subarray-sum.md`。

## 标签速查

| 标签 | 适用场景 |
|---|---|
| `#哈希表` | 两数之和、罗马数字转整数、基于哈希的 3Sum |
| `#字符串` | 字符串操作类题目 |
| `#滑动窗口` | 子数组/子串滑动窗口 |
| `#栈` | 有效括号、单调栈 |
| `#链表` | 链表操作 |
| `#二分` | 有序数组二分查找 |
| `#前缀和` | 前缀和、区间和查询 |
| `#差分数组` | 区间增减操作 |
| `#优先队列` | 堆排、Top-K |
| `#双指针` | 双指针技巧 |
| `#动态规划` | 通用 DP |
| `#背包问题` | 0/1 背包、完全背包 |
| `#数论` | 数论题 |
| `#区间调度` | 区间调度、不重叠区间 |
| `#计数` | 求方案数 |
| `#最优化` | 求最优值 |
| `#存在性` | 可行性判断 |
| `#二维` | 二维 DP 或矩阵 |
| `#容斥` | 容斥原理 |
| `#组合` | 组合问题 |

## 代码格式

- 代码块必须带语言标识：````python```` 或 ````c++````
- 早期版本常用 C++，后期版本常用 Python
- 错误版本可在代码中用 `# ❌` / `# ✅` 注释标记 bug

## 项目 Skill

项目级 skill 定义在 `.claude/skills/` 目录下，通过自然语言自动匹配触发：

| Skill | 触发关键词 | 功能 |
|-------|-----------|------|
| `practice` | 刷题、做第X题、没思路、卡住了 | 刷题引导 + 苏格拉底式学习 + 笔记整理 |
| `update-readme` | 更新README、更新统计、刷新进度 | 重新扫描全库，更新评估数据 |

无需斜杠命令，直接说需求即可。skill 根据 `description` 字段的语义自动匹配。
