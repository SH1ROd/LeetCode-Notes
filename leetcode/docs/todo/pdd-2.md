### 1. 多多的充电计划 -> **LeetCode 871. 最低加油次数** (或 POJ 2393 变形)

- **对应 LeetCode 题目**：[871. 最低加油次数 (Minimum Number of Refueling Stops)](https://leetcode.cn/problems/minimum-number-of-refueling-stops/)
    
- **重合算法**：`贪心` + `优先队列 (堆)`
    
- **结构匹配度解析**：
    
    - **核心逻辑**：LeetCode 871 是维护一个“最大堆”，遇到加油站就把油量丢进堆里，没油了就从堆顶（油量最多）“反悔”贪心加油。
        
    - **多多的充电计划**：逻辑如出一辙。把遇到的充电站的“价格和容量”放入**最小堆（按价格排序）**。当电量不足以跑到下一站时，从堆顶（价格最低的充电站）开始“购买”电量，直到足够跑到下一站。
        
    - **代码结构**：都需要遍历途径的站点 `for (int i=0; i<n; i++)`，在循环内部有一个 `while (current_power < required_power && !pq.isEmpty())` 的优先队列弹出逻辑。
        

### 2. 多多的扩容计划 -> **LeetCode 2528. 最大化城市的最小电量**

- **对应 LeetCode 题目**：[2528. 最大化城市的最小电量 (Maximize the Minimum Powered City)](https://leetcode.cn/problems/maximize-the-minimum-powered-city/)
    
- **重合算法**：`二分查找 (二分答案)` + `贪心` + `差分数组/滑动窗口`
    
- **结构匹配度解析**：
    
    - **核心逻辑**：LeetCode 2528 要求用最多 $k$ 座额外供电站（覆盖范围为 $2r+1$ 的区间），去最大化所有城市电量的最小值。这与“多多的扩容计划”完全同构：用最多 $m$ 个扩容包（覆盖范围为 $w$ 的区间），使得所有时间点的容量 $\ge$ 业务需求，并求最小的 $x$。
        
    - **代码结构**：
        
        1. 外层写一个 `left = 0, right = MAX` 的二分查找。
            
        2. 核心是一个 `check(mid)` 函数。
            
        3. `check` 函数内部：从左到右遍历，如果当前容量不够目标值，**贪心地在当前位置开启区间扩容包**，并使用**差分数组**（或滑动窗口记录影响量）来 $O(1)$ 维护后续区间受到的增益效果。
            

### 3. 多多的配送轨迹 -> **LeetCode 1658. 将 x 减到 0 的最小操作数** / **LeetCode 560. 和为 K 的子数组**

- **对应 LeetCode 题目**：[1658. 将 x 减到 0 的最小操作数](https://leetcode.cn/problems/minimum-operations-to-reduce-x-to-zero/)
    
- **重合算法**：`前缀和` + `哈希表` (2D坐标系下的目标和查找) / `双指针`
    
- **结构匹配度解析**：
    
    - **核心逻辑**：题意是删除中间的一段子串，将前缀和后缀拼接。这等价于：**找到一个连续子串，其包含的 U/D/L/R 数量恰好等于“总偏移量”减去“目标偏移量”**，并要求这个连续子串最短。
        
    - LeetCode 1658 的本质同样是“删除前缀和后缀使得其和为 target”，转化为“寻找中间一段连续子数组使得其和为 sum - target”。
        
    - **代码结构**：把 U/D/L/R 转化为二维坐标的向量 $(dx, dy)$。遍历字符串计算二维“前缀和”，并用一个 `HashMap<String/Pair, Integer>` 记录每种坐标状态最新出现的索引。当遍历到索引 $i$ 的坐标为 $(cur_x, cur_y)$ 时，去哈希表中查找是否存在对应的目标相对坐标，并以此更新最小子串长度。
        

### 4. 多多直播排行榜 -> **LeetCode 2146. 价格范围内最高排名的 K 样物品** / **LeetCode 2353. 设计食物评分系统**

- **对应 LeetCode 题目**：[2146. 价格范围内最高排名的 K 样物品](https://leetcode.cn/problems/k-highest-ranked-items-within-a-price-range/)
    
- **重合算法**：`多关键字自定义排序` + `哈希表模拟`
    
- **结构匹配度解析**：
    
    - **核心逻辑**：本题不包含复杂的算法推导，是一道经典的“业务模拟题”。难点在于**多维度的排序规则**（点赞 > 评论 > 时间 > 编号）以及**去重和映射**。
        
    - LeetCode 2146 同样要求依据四级规则（距离 > 价格 > 行坐标 > 列坐标）对对象进行严格的自定义排序。
        
    - **代码结构**：
        
        1. 使用 `HashMap<UserID, Item>` 遍历输入。当遇到重复主播时，直接用题目给定的四级比较逻辑覆盖更优的值（主播去重）。
            
        2. 将哈希表的 values 倒出为列表/数组，重用上述排序逻辑调用 `Collections.sort()`。
            
        3. 遍历排序后的列表，用另一个 `HashMap<Item_ID, Rank>` 记录最终排名，以此 $O(1)$ 回答 $q$ 次查询。
