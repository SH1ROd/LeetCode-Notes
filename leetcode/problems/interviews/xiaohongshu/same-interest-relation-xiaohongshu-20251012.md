# 题目解析

- **平台**：小红书社交推荐项目  
- **目标**：基于用户的行为习惯分数，挖掘“同好”关系（即兴趣相似的用户）
## 数据说明

- 数据库中共有 `n` 个用户  
- 每个用户有一个 **行为习惯分数** `a_i`  
- 系统需要处理 `m` 次查询

---

## “同好”关系的定义

用户 **i** 和用户 **j** 构成 “同好” 关系，当且仅当满足以下条件之一：

```

a_i 能被 a_j 整除
或者
a_j 能被 a_i 整除

```

也就是：

```

a_i | a_j  或  a_j | a_i

```

---

## 查询任务

每次查询给定一个额外用户的分数 `x`，  
统计数据库中有多少个用户与该用户构成“同好”关系。

---

## 输入输出格式

**输入：**
1. `n, m` —— 用户数量与查询次数  
2. `a1, a2, ..., an` —— 数据库中用户的分数  
3. `m` 行，每行一个 `x` —— 查询用户的分数  

**输出：**
- 对每个查询，输出一个整数，表示满足“同好”关系的用户数。

---

## 数据范围

```

1 ≤ n, m ≤ 5 × 10^5
1 ≤ a_i, x ≤ 5 × 10^5

```
