### 第一部分：这道题你到底错在哪了？（破除迷障）

这道题你之所以会卡住，有**两个直接原因**：

**1. 被题目的名字“骗”了**

题目叫《组合总和 Ⅳ》，但你在描述里仔细看这句：“请注意，顺序不同的序列被视作不同的组合”。

在数学和算法的严格定义里，**看重顺序的叫“排列”（Permutation），不看重顺序的才叫“组合”（Combination）**。这道题披着“组合”的皮，实际上要你求的是“全排列”。你的潜意识被题目名字带偏了，以为只要挑出数字就行。

**2. 原代码抹杀了“顺序”的可能性**

你之前写的代码是：

Python

```
for i in nums:
    for j in range(i, target + 1):
```

这个代码的运行轨迹是：先拿数字 1 去凑所有的和，再拿数字 2 去凑所有的和，最后拿数字 3。

这就导致了一个致命结果：**数字是被“强制按顺序”塞进背包的。** 它能凑出 `(1, 1, 2)`，因为它先选了 1，后选了 2。但它**永远不可能**凑出 `(2, 1, 1)` 或 `(1, 2, 1)`，因为当循环走到数字 2 的时候，关于数字 1 的决策已经彻底结束了，代码无法“回头”再去选 1。

所以，你算出来的 4，是纯粹的“组合数”，而题目要求的是包含不同顺序的 7 个“排列数”。

### 第二部分：嵌套循环的“道” —— 时间轴与菜单

抛开具体的 `nums`、`target`、`dp`，泛化到所有包含“选择”的两层循环动态规划中，你只需要在脑子里建立这样一个极其底层的直觉模型：

在所有的两层循环 DP 中：

- **外层循环 = 时间轴（一旦走过，不可回头）**
    
- **内层循环 = 当期菜单（在这个时间点上，你能拥有的所有选择）**
    

#### 场景 A：外层物品，内层容量（你原来的错误思路）

- **时间轴（外层）：** 物品 1 阶段 $\rightarrow$ 物品 2 阶段 $\rightarrow$ 物品 3 阶段。
    
- **底层物理意义：** 你的时间轴是跟着**物品**走的。当你在“物品 1 阶段”时，你随便怎么选物品 1；但是，一旦时间轴推进到了“物品 2 阶段”，物品 1 的大门就**永远关闭**了！你再也回不去选物品 1 了。
    
- **结果：** 因为时间单向流动，你只能先选 1，再选 2，再选 3。所有的选择被强制按物品的遍历顺序排列，抹杀了顺序的多样性。这就是**求组合数**。
    

#### 场景 B：外层容量，内层物品（这道题的正确思路）

- **时间轴（外层）：** 容量为 1 的阶段 $\rightarrow$ 容量为 2 的阶段 $\rightarrow$ 容量为 3 的阶段。
    
- **底层物理意义：** 你的时间轴是跟着**目标容量**走的。在凑每一个目标容量的阶段里，你的“菜单（内层循环）”上**永远摆着所有的全量物品**。
    
- **结果：** 在凑容量 2 的时候，你可以点“物品 1”这道菜；到了凑容量 3 的时候，时间来到了新阶段，菜单刷新了，你**依然**可以点“物品 1”！你可以先选 2，再选 1；也可以先选 1，再选 2。只要在当期菜单上，你随时可以吃回头草。所有的顺序都被允许，这就是**求排列数**。
    

### 总结你的泛化判断依据

以后拿到任何类似的题目，不要去背什么“完全背包套路”，只问自己一个核心问题：

**“这道题，允许我吃回头草吗？”（即：(1, 2) 和 (2, 1) 算不算两种不同的结果？）**

- **如果算同一种（不准吃回头草）：** 外层循环必须是**物品**。用时间轴强行锁死物品的出场顺序。
    
- **如果算不同种（允许吃回头草，顺序很重要）：** 外层循环必须是**容量（目标）**。让每次凑目标时，所有物品的菜单都对你全量敞开。
    

---

在动态规划的“完全背包”模型（物品可以无限次使用）中，确实有一类问题，**无论是外层遍历物品还是外层遍历容量，算出来的答案都是完全一样的！**

那么，凭什么有的题必须严格区分内外循环，有的题却可以随便换？这背后的“道”是什么？

### 核心心法：你要的是“方案数”，还是“最值”？

记住这个极简的判定标准：

- 如果题目求的是**“有多少种方法/方案数”** $\rightarrow$ **必须严格区分**内外循环（因为顺序关乎到你是求组合还是求排列）。
    
- 如果题目求的是**“最大值 / 最小值 / 最少个数”** $\rightarrow$ **内外循环随便换**（因为顺序根本不影响最终的“极值”）。
    

#### 为什么求“最值”时，内外循环可以随便换？

回到我们刚刚的“时间轴与菜单”理论：

- 先物品，后容量（组合视角）：算出来的是 `[1, 2]`。
    
- 先容量，后物品（排列视角）：算出来的是 `[1, 2]` 和 `[2, 1]`。
    

如果题目问：“凑出 3 的**最少硬币数量**是多少？”

不管你是按组合视角拿到的 `[1, 2]`，还是按排列视角拿到的 `[2, 1]`，你手里捏着的硬币总数**都是 2 枚**！

求最值（Max/Min）时，**顺序的差异被“最值操作”天然地抹平了**。无论你怎么换菜的顺序，这顿饭的总卡路里（最大价值）或者最少盘子数（最少硬币）是一模一样的。

### 举个经典的例子（内外循环皆可）

**LeetCode 322. 零钱兑换**

> 题目：给你几种面额的硬币（无限量），和一个总金额。求凑成总金额所需的**最少硬币个数**。

这道题求的是“最少个数（最小值）”，所以下面两种写法**完全等价，都能通过**：

**写法 A（先物品，后容量 —— 组合视角）：**

Python

```
# 逻辑：我先固定用 1 块钱硬币去更新所有金额的最少个数，再用 2 块钱硬币去更新...
for coin in coins:
    for j in range(coin, amount + 1):
        dp[j] = min(dp[j], dp[j - coin] + 1)
```

**写法 B（先容量，后物品 —— 排列视角）：**

Python

```
# 逻辑：我要凑金额 1，试试所有硬币；我要凑金额 2，再试试所有硬币...
for j in range(1, amount + 1):
    for coin in coins:
        if j >= coin:
            dp[j] = min(dp[j], dp[j - coin] + 1)
```

### ⚠️ 必须要防范的一个“刺客”：0-1 背包

前面说的“随便换”，前提是**完全背包（物品无限使用）且求最值**。

如果你遇到的是 **0-1 背包（每个物品只能用一次）**，并且你为了省内存，把 `dp` 数组压缩成了一维 `dp[j]`，那么**绝对不能颠倒内外循环**！

0-1 背包的一维写法必须是：

**外层物品，内层容量（且容量必须倒序遍历）**

如果你把容量放到外层，或者把内层容量改成正序，它会立刻因为“吃了回头草”（重复使用了同一个物品）而变成完全背包，导致答案全错。

### 一张表总结你的 DP 循环直觉

你在脑子里存下这张表，以后碰到背包问题，直接“照妖镜”伺候：

|**题目特征**|**目标**|**循环顺序规则**|
|---|---|---|
|**完全背包** (物品无限)|求 **组合数** (不看顺序)|**必须** 外层物品，内层容量|
|**完全背包** (物品无限)|求 **排列数** (看重顺序)|**必须** 外层容量，内层物品|
|**完全背包** (物品无限)|求 **最大/最小值**|**随便换**，内外谁先都一样|
|**0-1 背包** (物品一次)|无论求什么 (一维 DP下)|**必须** 外层物品，内层倒序容量|
