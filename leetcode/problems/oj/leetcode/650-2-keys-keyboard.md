```python
class Solution:
    def minSteps(self, n: int) -> int:
        dp = [float('inf')] * (n + 1)
        dp[1] = 0
        for i in range(2, n + 1):
            for j in range(1, i + 1): # 右边界暴力
                if i % j == 0:
                    dp[i] = min(dp[i], dp[j] + (i // j))
        return dp[n]
```

### 第一阶段：抓住直觉，但漏掉操作代价

- **你的初步尝试：** `if j % 2 == 0: dp[j] = min(dp[j], dp[j // 2])`
    
- **闪光点：** 你本能地察觉到这题和“倍数/因数”有关，想到了长度可以翻倍。
    
- **存在的盲区：** 从 `j // 2` 变成 `j`，不是无代价的。它需要先“全部复制”（1步），再“粘贴”（1步），总共消耗 2 步。
    
- **改进策略：** 状态转移应该加上具体的操作步骤代价，即 `dp[j] = min(dp[j], dp[j // 2] + 2)`。
    

### 第二阶段：推导通式，但在边界上栽了跟头

- **你的进阶代码：** `for j in range(1, int(n ** 0.5) + 1): dp[i] = min(dp[i], dp[j] + (i // j))`
    
- **闪光点：** 你成功把只看 `2` 的特例，泛化成了任何因数 `j` 的通式，写出了完美的核心状态转移方程！
    
- **存在的盲区（两个Bug）：**
    
    1. **目标错位：** 你的循环范围用了全局的 `n`，但其实求 `dp[i]` 时，应该只看当前长度 `i` 的因数。
        
    2. **漏掉大因数：** 开平方根找因数是对的，但因数是成对的。比如 $i = 12$ 时，平方根是 $3.46$。你找到了小因数 `2` 和 `3`，却忘了把它们对应的大因数 `6` 和 `4` 也拿来更新状态。
        
- **改进策略：** 要么扩大遍历范围，要么在循环里把“大因数”也一起算上。
    

### 第三阶段：回归暴力兜底（你现在的版本）

- **你的最终代码：** `for j in range(1, i + 1):`
    
- **闪光点：** 逻辑无懈可击！你通过遍历所有的数字，确保绝不会漏掉任何一个因数。这个版本能顺利通过测试（AC）。
    
- **存在的盲区（冗余计算）：**
    
    - 当 $j > i // 2$ 且 $j < i$ 时，`j` 绝对不可能是 `i` 的因数（比如 10 的因数不可能是 6, 7, 8, 9），这部分循环在做无用功。
        
    - 当 $j = i$ 时，计算毫无意义（复制当前所有字符，但不粘贴，长度不变，还白费操作）。
        
- **改进策略：** 进行“剪枝”，去掉不可能的范围。
    

---

### 第四阶段：优化剪枝（我的两个方法）

既然我们知道了第三阶段的代码在做无用功，我们就可以顺理成章地对循环边界进行裁剪：

**方法一：折半法（最符合直觉的优化）**

一个数字最大的真因数，永远不会超过它自己的一半。

- **做法：** 把 `range(1, i + 1)` 直接改成 `range(1, i // 2 + 1)`。
    
- **效果：** 砍掉了一半的无效遍历，代码依然简单易懂。
    

**方法二：平方根双向更新（极致性能的优化）**

如果你还想更快，回忆一下我们在第二阶段犯的错误：开平方根会漏掉大因数。那我们把它补上不就好了？

- **做法：** 循环缩减到 `range(1, int(i ** 0.5) + 1)`。当找到一个能够整除的小因数 `j` 时，`i // j` 就是那个大因数。
    
- **核心代码：**
    
    Python
    
    ```
    if i % j == 0:
        # dp[j] + (i // j) 是从小因数转移过来的代价
        # dp[i // j] + j 是从大因数转移过来的代价
        dp[i] = min(dp[i], dp[j] + (i // j), dp[i // j] + j)
    ```
    
- **效果：** 时间复杂度从 $O(n^2)$ 骤降到 $O(n \sqrt{n})$。
    

把这四个阶段串起来看，你就明白优化是怎么一步步被“逼”出来的了。这种思维推演的价值，远大于死记硬背一个最优解。
